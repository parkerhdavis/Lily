# Managing Questionnaires

Questionnaire definitions control the structure and content of the client intake questionnaire — which sections appear, what questions are asked, and how those questions map to template variables.

---


## Accessing the Questionnaire Editor

From the **Pipeline**, navigate to the **Client Setup** section and select a questionnaire definition to edit, or create a new one.

---


## Questionnaire Structure

A questionnaire definition consists of:

- **Tabs** — Top-level groupings (e.g., "Client Info," "Client Contacts," "Assignments & Decisions")
- **Sections** — Collapsible groups of related questions within each tab (e.g., "Client Information," "Healthcare POA")
- **Questions** — Individual fields within each section

### Question Types

Each question has a type that determines how it appears in the client questionnaire:

| Type | Appearance | Maps to |
|---|---|---|
| **Text** | Text input field | Replacement variable |
| **Conditional** | Checkbox / toggle | Conditional variable |
| **Contact-role** | Contact dropdown | Contact binding + variables |

---


## Creating a Questionnaire

1. In the Pipeline's Client Setup section, click **Create New**
2. Give the questionnaire a name
3. Add sections and questions using the editor
4. Save when you're done

### Setting the Active Questionnaire

One questionnaire is designated as **active** — this is the one used for new clients. To change which questionnaire is active, select it in the list and mark it as active.

---


## Editing a Questionnaire

Open an existing questionnaire definition to modify its sections and questions. Changes are saved with an incremented version number, so there's a clear history of modifications.

### Duplicating

If you want to create a variant of an existing questionnaire (e.g., for a different practice area), use the **Duplicate** option. This creates a copy that you can rename and modify independently.

---


## Versioning

Each time you save a questionnaire definition, its version number increments automatically. When a client's questionnaire is opened, the `.lily` file records the questionnaire ID and version, creating an audit trail of which intake form was used.

This means you can update a questionnaire definition without affecting clients who were already processed with an earlier version — their `.lily` files still reference the version they used.

---


## Default Questionnaire

On first launch, Lily seeds a default questionnaire definition (currently "Elder Law Estate Planning") with sections and questions tailored for that practice area. You can modify this default, duplicate it as a starting point for other practice areas, or create entirely new definitions from scratch.
