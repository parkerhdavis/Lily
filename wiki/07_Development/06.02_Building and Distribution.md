# Building and Distribution

How to build Lily for release.

---


## Build Commands

```bash
make build            # Build for current platform (auto-detects OS)
make build-linux      # Linux installers (.deb, .rpm, AppImage)
make build-windows    # Windows installers (.msi, .exe)
make build-macos      # macOS installers (.dmg, .app)
```

Build output is placed in the `target/release/bundle/` directory (or the platform-specific target directory for cross-compilation).

---


## Platform Notes

### Linux

Produces three installer formats:
- **AppImage** — Universal binary, no installation required
- **.deb** — For Debian/Ubuntu-based distributions
- **.rpm** — For Fedora/RHEL-based distributions

### Windows

Produces:
- **.msi** — Standard Windows installer
- **.exe** — NSIS installer

### macOS

Produces:
- **.dmg** — Disk image with drag-to-install
- **.app** — Application bundle

---


## Important Notes

- `make build` is slow — only run it when you need release artifacts
- For development validation, use `make lint`, `make typecheck`, and `make test` instead
- This is a desktop application, not a web app — bundle size optimizations are generally unnecessary
