# Developer Quick Start

This guide gets you up and running with Lily development.

---


## Prerequisites

- **Rust** toolchain (install via [rustup](https://rustup.rs/))
- **Bun** runtime and package manager (install via [bun.sh](https://bun.sh/))
- **Platform dependencies:**
	- Linux: WebKitGTK 4.1+, GTK 3+, and related dev packages
	- Windows: WebView2 (usually pre-installed on Windows 10+)
	- macOS: Xcode command-line tools

---


## Setup

```bash
git clone https://github.com/parkerhdavis/Lily.git
cd Lily
make setup
```

`make setup` installs all frontend (Bun) and backend (Rust) dependencies.

---


## Development

```bash
make dev              # Full Tauri dev server (frontend + Rust hot-reload)
make dev-frontend     # Bun dev server only (rapid UI iteration, no Tauri shell)
```

`make dev` launches the full desktop app with hot-reloading for both the React frontend and the Rust backend. `make dev-frontend` is faster to start and useful when you're only working on UI.

---


## Quality Checks

```bash
make lint             # Biome check + Rust clippy
make lint-fix         # Biome check with auto-fix
make format           # Biome format + rustfmt
make typecheck        # TypeScript type checking
make test             # Cargo test + Bun test
```

Run `make lint`, `make typecheck`, and `make test` before submitting changes.

## Tech Stack

| Layer | Technology |
|---|---|
| Desktop framework | Tauri 2.0 (Rust) |
| Frontend | React 19 + TypeScript |
| Bundler / runtime | Bun |
| Styling | Tailwind CSS 4 + daisyUI 5 |
| State management | Zustand 5 |

## Project Structure

```
Lily/
├── backend/src/          # Rust backend (Tauri commands)
│   ├── main.rs           # Tauri setup, command registration
│   ├── docx_ops.rs       # Document operations (copy, extract, replace, preview)
│   ├── lily_file.rs      # .lily project file management
│   ├── settings.rs       # User preferences persistence
│   └── questionnaire.rs  # Questionnaire definition CRUD
├── frontend/src/         # React frontend
│   ├── components/       # UI components
│   ├── stores/           # Zustand state management
│   ├── types/            # TypeScript type definitions
│   └── styles/           # CSS
├── resources/            # App icons, seed data
└── wiki/                 # User-facing documentation (you are here)
```

## Coding Standards

- **Tab indentation** (enforced by Biome)
- **Double quotes** for strings
- **Functional components** with hooks
- **daisyUI components** with semantic color utilities (`bg-base-100`, `text-primary`)
- **Never hard-code colors** — use CSS variables or daisyUI classes for theme support
- **Path aliases** — use `@/` prefix for imports from `src/`
