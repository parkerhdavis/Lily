# The Client Hub

The Client Hub is your home base for a client. It's where you see all of a client's documents, manage their information, and navigate to other parts of the workflow.

---


## Layout

The Client Hub has a two-panel layout:

### Left Panel — Documents

The left side shows everything related to the client's documents:

- **Questionnaire Card** — At the top, a card shows the client's questionnaire progress (e.g., "12 / 35 fields completed"). Click it to open the questionnaire.
- **Document List** — Below that, all documents in the client folder are listed with their names and source templates. Click a document to open it in the Variable Editor.
- **Add New Document** — A button at the bottom opens the Template Picker to add another document.

If you have more than a few documents, a search filter appears to help you find the one you need.

### Right Panel — Client Info

The right side shows client-level information:

- **Variables** — All shared variable values for this client. You can edit them directly here — changes save automatically when you click away from a field.
- **Add/Remove Variables** — You can manually add new variables or remove ones you no longer need.
- **Manage Contacts** — A button that toggles the panel to show the Contact Manager, where you can add and edit people associated with the client.

---


## Document Status Tracking

Each document can be assigned a status that tracks its progress through the workflow:

| Status | Description |
|---|---|
| **Not Started** | Template has been added but no work has begun |
| **Drafting** | Document is being filled in |
| **Reviewing** | Document is ready for review |
| **Complete** | Document has been finalized |
| **Executed** | Document has been signed/executed |

These statuses are also visible in the [Clients Hub](wiki/02_Client_Management/01.00_Working%20with%20Clients.md), which shows an overview of all clients and their document progress.

---


## Document Versioning

You can create a dated copy of any document by using the **New Version** option. This creates a copy with today's date appended to the filename (e.g., `Trust Agreement-20260325.docx`), preserving the original while letting you work on an updated version.

---


## Navigation

From the Client Hub, you can:

| Action | How |
|---|---|
| Open a document | Click it in the document list |
| Open the questionnaire | Click the Questionnaire card |
| Add a new document | Click **Add New Document** |
| Manage contacts | Click **Manage Contacts** in the right panel |
| Return to the Lily Hub | Click the **Back** button in the header, or use the grid menu in the header |
| Go to Pipeline or Settings | Use the grid menu in the header |
| Open the client folder in your file manager | Click the folder icon in the status bar |

The **Back** button from the Client Hub returns you to the Lily Hub and clears the current client context. The grid menu lets you jump to other areas of the app without losing client state.

---


## Working with the Document List

### Opening a Document

Click any document in the list to open it in the Variable Editor. Your variable values are loaded from the shared client pool, and the document preview renders with the current values.

### Document Status

Documents that were created from a template show the source template name. This helps you see at a glance which templates have been used for the client.

---


## Editing Variables Directly

The right panel lets you edit any variable value directly, without opening a specific document. This is useful for filling in basic client information (name, address, phone) that's shared across all documents.

Changes save to the `.lily` file automatically when you move focus away from the field.
