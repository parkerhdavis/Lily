# Working with Clients

Lily organizes work by **client folder** — one folder per client, containing all of that client's documents and data. This mirrors how most law offices already organize their files.

---


## Client Folders

A client folder is just a regular directory on your computer. It might be something like:

```
Clients/
├── Doe, Jane/
│   ├── Doe, Jane.lily
│   ├── Financial POA - Doe.docx
│   └── Healthcare POA - Doe.docx
├── Smith, John/
│   ├── Smith, John.lily
│   ├── Living Will - Smith.docx
│   └── Engagement Letter - Smith.docx
```

When you open a client folder in Lily for the first time, Lily creates a **project file** (`.lily`) in that folder. This file stores the client's variable values, contacts, and document metadata. It's created and managed automatically — you generally don't need to interact with it directly.

---


## Opening a Client

From the **Lily Hub** (home screen), click **Open Client Folder** and select the client's directory. If the folder already has a `.lily` file, Lily picks up where you left off. If not, Lily creates one and you start fresh.

You can also switch between clients at any time using the grid menu (⊞) in the header to return to the Lily Hub.

---


## Shared Variables

One of Lily's key features is that variable values are **shared across all documents** for the same client. When you fill in `{Client Full Name}` in one document, that value is available in every other document for that client.

This means you typically enter a client's basic information once — through the questionnaire or while working on the first document — and it flows into all subsequent documents automatically.

---


## The .lily File

The `.lily` file is a project file that lives in the client folder. It stores:

- **Variable values** — All the information you've entered for this client (names, addresses, dates, etc.)
- **Contact records** — People associated with the client and their role assignments
- **Document list** — Which templates were used and when documents were created
- **Questionnaire reference** — Which questionnaire definition was used for intake

The `.lily` file uses a custom extension so it can eventually be associated with Lily at the OS level — meaning you could double-click it to open Lily directly to that client.

**Important:** The `.lily` file is plain JSON under the hood. While you shouldn't need to edit it by hand, it's not a black box — you can inspect it in any text editor if needed.

---


## Client Data Export and Import

You can export a client's data (variables, contacts, and document metadata) to a JSON file for backup or transfer:

1. In the **Client Hub**, click the menu in the header
2. Select **Export** to save the data, or **Import** to load data from a previously exported file

When importing, the imported data is merged with any existing data. If there's a conflict (e.g., the same variable has different values), the imported value wins.
