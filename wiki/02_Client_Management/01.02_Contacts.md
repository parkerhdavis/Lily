# Contacts

Contacts are people associated with a client — agents, trustees, beneficiaries, personal representatives, and so on. Lily's contact system lets you store their information once and assign them to roles, so their details automatically populate the right variables in your documents.

---


## The Contact Manager

Access the Contact Manager from the **Client Hub** by clicking **Manage Contacts** in the right panel. From here you can add, edit, and delete contacts.

### Contact Fields

Each contact stores:

| Field | Description |
|---|---|
| Full Name | The contact's complete name |
| First Name | First name (used for some variable mappings) |
| Last Name | Last name |
| Relationship | How they relate to the client (e.g., "Son," "Spouse") |
| Phone | Phone number |
| Email | Email address |
| Address | Street address |
| City | City |
| State | State |
| Zip | ZIP code |

Email, phone, and zip fields have basic format validation — Lily will show a hint if something looks off, but won't prevent you from saving.

---


## Contact Roles

The real power of contacts comes from **role assignment**. Many legal documents reference people by their role — "Healthcare POA Agent," "Alternate Trustee," "Personal Representative," and so on.

When you assign a contact to a role, Lily automatically maps the contact's information to the corresponding variables. For example, assigning Jane Smith as the "Healthcare POA Agent" might fill in:

- `{Healthcare POA Agent Name}` → "Jane Smith"
- `{Healthcare POA Agent Phone}` → "(555) 123-4567"
- `{Healthcare POA Agent Address}` → "123 Main St"

This mapping is defined by the variable names in your templates. When a contact is assigned to a role, their property values are written directly into the shared variable pool — the document templates don't need to know anything about contacts.

---


## Assigning Contacts to Roles

Contacts are typically assigned to roles through the **Questionnaire**, where contact-role questions present a dropdown of available contacts. You can also select "Other" to enter values manually without using a contact record.

In the **Variable Editor**, variables that are filled by a contact binding show a link icon and the role name, so you can see at a glance which values come from contact assignments.

---


## Deleting a Contact

When you delete a contact, Lily warns you if that contact is bound to any roles. The variable values that were populated from the contact remain in place (they don't disappear), but they'll no longer update automatically if you change the contact's information.
