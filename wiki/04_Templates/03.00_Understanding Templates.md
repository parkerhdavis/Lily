# Understanding Templates

Templates are the foundation of Lily's document workflow. They're standard Word documents (`.docx`) with placeholder variables that Lily fills in with client-specific information.

---


## What a Template Looks Like

A template is a `.docx` file — created in Word, LibreOffice, Google Docs, or any word processor that saves to `.docx` format. The key difference from a regular document is that it contains **variables**: placeholders written in curly braces.

For example, a simple engagement letter template might contain:

```
Dear {Client Full Name},

This letter confirms that our firm has been engaged to provide
{Scope of Representation} services on your behalf.

Our fee for this engagement is {Fee Amount}.
```

When a client document is created from this template, Lily extracts those variables and presents them as a form. Once filled in, the placeholders are replaced with the actual values.

---


## Variable Syntax

### Replacement Variables

The basic syntax is a name inside curly braces:

```
{Variable Name}
```

Variable names can contain letters, numbers, spaces, and basic punctuation. They're **case-insensitive** — `{Client Full Name}` and `{client full name}` are treated as the same variable.

**Case-aware replacement:** While variables are matched case-insensitively, the casing of each occurrence controls how the value is displayed:

| Placeholder in template | Value entered | Result |
|---|---|---|
| `{Client Full Name}` | Jane Doe | Jane Doe |
| `{CLIENT FULL NAME}` | Jane Doe | JANE DOE |
| `{client full name}` | Jane Doe | jane doe |

This lets you use the same variable in different contexts — title case in the salutation, all caps in a header — without entering multiple values.

### Conditional Variables

Conditional variables let a single placeholder resolve to one of two text options based on a yes/no toggle:

```
{Label ?? text if yes :: text if no}
```

For example:

```
{Client Is Single ?? I, the undersigned individual :: We, the undersigned}
```

In the Variable Editor, this appears as a checkbox labeled "Client Is Single." The document shows one text or the other based on the checkbox state. Either branch can be empty:

```
{Has Alternate Agent ?? The alternate agent is {Alternate Agent Name}. ::}
```

**Note:** The doubled operators (`??` and `::`) are intentional — single `?` and `:` can appear freely in the output text without being misinterpreted.

## Where Templates Live

Templates are stored in a folder on your computer that you configure in [App Settings](wiki/06_Settings/05.00_App%20Settings.md). You can organize them in subfolders however you like — Lily's Template Picker mirrors your folder structure.

A typical setup might look like:

```
Templates/
├── Estate Planning/
│   ├── Financial POA.docx
│   ├── Healthcare POA.docx
│   └── Living Will.docx
├── Family Law/
│   ├── Engagement Letter.docx
│   └── Retainer Agreement.docx
└── General/
    └── Cover Letter.docx
```

---


## How Templates Are Used

When you select a template for a client:

1. Lily **copies** the template into the client folder — the original template is never modified
2. The copy is opened in the Variable Editor
3. Variables are extracted from the document and shown as form fields
4. When you save, variable values are written into the copy

The template itself stays pristine. You can use it for as many clients as you need.

---


## Formatting

Templates preserve all standard Word formatting. Variables inherit the formatting of the text around them — if a placeholder is inside bold text, the filled-in value will be bold too. This means you format your templates exactly how you want the final document to look, and the variable replacement doesn't disrupt it.

---


## Split Variable Handling

Word sometimes splits text across multiple internal XML elements — for example, `{Client Name}` might end up as `{` in one run, `Client Name` in another, and `}` in a third. This happens when Word applies spelling corrections, formatting changes, or other internal processing.

Lily handles this automatically. A pre-processing step merges split variables back together before extraction, so you don't need to worry about it. If a variable isn't being detected, try deleting the placeholder and retyping it in one go.
