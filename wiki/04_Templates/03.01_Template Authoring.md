# Template Authoring

Lily includes a built-in template authoring tool that lets you create templates visually, without manually typing variable syntax into a Word document.

---


## How It Works

Instead of writing `{Variable Name}` placeholders by hand in Word, you can:

1. Start with a **flat document** — a regular `.docx` with the text you want but no variables yet
2. Open it in Lily's template authoring mode
3. Select text in the preview and replace it with a variable

This approach is especially useful when converting existing documents into templates. Take a completed document, open it in the authoring tool, and turn the client-specific parts into variables.

---


## Creating a Template

1. Place a `.docx` or `.dotx` file (without any variables) in your templates folder
2. Open the **Pipeline** from the grid menu (⊞)
3. Browse to the document and open it in authoring mode

### Inserting a Variable

1. Select text in the document preview
2. A prompt appears asking for the variable name
3. Enter the name (e.g., "Client Full Name") and confirm
4. The selected text is replaced with the variable placeholder

If the text you selected appears multiple times in the document, Lily asks which occurrence(s) you want to replace — so you can be precise about which instances become variables.

### Removing a Variable

To remove a variable and restore the original text:

1. Click the variable in the preview
2. Select the remove/restore option
3. The original text is put back in place

---


## Variable Schema Auto-Save

When you insert a variable through the authoring tool, Lily automatically creates a [variable schema](wiki/04_Templates/03.02_Variable%20Schemas.md) entry for it. This means type definitions, required flags, and help text are saved alongside the template.

---


## Tips

- **Start from a completed document** — It's often easier to take a finished document and replace the client-specific parts with variables, rather than writing a template from scratch.
- **Use clear, consistent names** — `{Client Full Name}` is better than `{name}`. Clear names make the variable form easier to fill in and help with variable sharing across documents.
- **Check the preview** — After inserting variables, review the preview to make sure the formatting looks right.
