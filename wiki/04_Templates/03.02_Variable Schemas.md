# Variable Schemas

Variable schemas let you define metadata about each variable in a template — its type, whether it's required, a default value, and help text. This makes the Variable Editor smarter and more helpful for the person filling in the document.

---


## What Schemas Do

Without a schema, every variable is a plain text field. With a schema, you can specify:

| Property | Effect in the Variable Editor |
|---|---|
| **Type** | Controls the input widget (text field, date picker, currency field, or conditional toggle) |
| **Required** | Shows a "Required" badge on the variable |
| **Default** | Pre-fills the field with a default value |
| **Help Text** | Shows a description or hint below the field |
| **Date Format** | For date variables, specifies the display format (e.g., `MM/DD/YYYY`) |

---


## Variable Types

### Text (default)

A standard text input. This is the default if no schema is defined.

### Date

Shows a date picker widget. Useful for variables like `{Date of Signing}` or `{Date of Birth}` where you want a consistent date format. You can optionally set a `date_format` (e.g., `"MM/DD/YYYY"`) in the schema entry.

### Currency

Shows a text field with a `$` prefix. Useful for variables like `{Fee Amount}` or `{Asset Value}`.

### Conditional

For variables that use the `{Label ?? "true text" :: "false text"}` syntax. Shows a toggle instead of a text field.

---


## How Schemas Are Stored

Schemas are stored in a sidecar `.lily` file alongside the template, with a special `lily_type: "template-schema"` marker. The schema file uses the template's name with a `.lily` extension — for example, if your template is `Financial POA.docx`, the schema file would be `Financial POA.lily`.

These schema files are created automatically when you use the [Template Authoring](wiki/04_Templates/03.01_Template%20Authoring.md) tool. You can also create or edit them by hand — they're plain JSON.

---


## Schema File Format

A schema file looks like this:

```json
{
    "lily_type": "template-schema",
    "template_filename": "Financial POA.docx",
    "variables": {
        "Client Full Name": {
            "var_type": "text",
            "required": true,
            "help": "The client's full legal name"
        },
        "Date of Signing": {
            "var_type": "date",
            "required": true,
            "date_format": "MM/DD/YYYY"
        },
        "Fee Amount": {
            "var_type": "currency",
            "default": "500.00"
        }
    }
}
```

### Field Reference

| Field | Required | Description |
|---|---|---|
| `lily_type` | Yes | Must be `"template-schema"` |
| `template_filename` | Yes | The filename of the template this schema applies to (e.g., `"Financial POA.docx"`) |
| `variables` | Yes | Map of variable names to their schema entries |

Each variable entry supports:

| Field | Default | Description |
|---|---|---|
| `var_type` | `"text"` | One of: `"text"`, `"date"`, `"currency"`, `"conditional"` |
| `required` | `false` | Whether the variable is required |
| `default` | — | Default value to pre-fill |
| `help` | — | Help text shown below the field |
| `date_format` | — | Date display format (only for `"date"` type) |

---


## When You Don't Need Schemas

Schemas are entirely optional. Templates work fine without them — every variable simply shows as a text field. Schemas are a convenience layer that improves the editing experience, especially for templates that are used frequently or by multiple people.
