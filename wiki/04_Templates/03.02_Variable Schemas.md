# Variable Schemas

Variable schemas let you define metadata about each variable in a template — its type, whether it's required, a default value, and help text. This makes the Variable Editor smarter and more helpful for the person filling in the document.

---


## What Schemas Do

Without a schema, every variable is a plain text field. With a schema, you can specify:

| Property | Effect in the Variable Editor |
|---|---|
| **Type** | Controls the input widget (text field, date picker, or currency field) |
| **Required** | Shows a "Required" badge on the variable |
| **Default** | Pre-fills the field with a default value |
| **Help Text** | Shows a description or hint below the field |

---


## Variable Types

### Text (default)

A standard text input. This is the default if no schema is defined.

### Date

Shows a date picker widget. Useful for variables like `{Date of Signing}` or `{Date of Birth}` where you want a consistent date format.

### Currency

Shows a text field with a `$` prefix. Useful for variables like `{Fee Amount}` or `{Asset Value}`.

---


## How Schemas Are Stored

Schemas are stored in a sidecar `.lily` file alongside the template, with a special `lily_type: "template-schema"` marker. For example, if your template is `Financial POA.docx`, the schema file would be `Financial POA.docx.lily`.

These schema files are created automatically when you use the [Template Authoring](wiki/04_Templates/03.01_Template%20Authoring.md) tool. You can also create or edit them by hand — they're plain JSON.

---


## Schema File Format

A schema file looks like this:

```json
{
    "lily_type": "template-schema",
    "variables": {
        "Client Full Name": {
            "type": "text",
            "required": true,
            "help": "The client's full legal name"
        },
        "Date of Signing": {
            "type": "date",
            "required": true
        },
        "Fee Amount": {
            "type": "currency",
            "default": "500.00"
        }
    }
}
```

---


## When You Don't Need Schemas

Schemas are entirely optional. Templates work fine without them — every variable simply shows as a text field. Schemas are a convenience layer that improves the editing experience, especially for templates that are used frequently or by multiple people.
