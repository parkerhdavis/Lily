# Saving and Renaming

Once you've filled in variables, you'll want to save the document and optionally rename it.

---


## Saving

Press `Ctrl+S` (or `Cmd+S` on Mac) to save. This does two things:

1. **Writes variable values** into the `.docx` file — replacing the placeholders with your actual text
2. **Updates the `.lily` file** — saving all variable values so they're available for other documents

An **"Unsaved" badge** appears in the Variable Editor header whenever you have changes that haven't been saved. It disappears once you save.

### What Gets Saved

- All variable values (replacement text, conditional states)
- Contact bindings and role assignments
- Questionnaire notes

Variable values are saved at the client level, not per-document. So saving in any document updates the shared pool that all documents draw from.

---


## Renaming a Document

You can rename a document directly from the Variable Editor:

1. **Double-click** the document title in the header
2. Type the new name
3. Press **Enter** to confirm, or **Escape** to cancel

Lily renames the actual `.docx` file on disk and updates the `.lily` file to reflect the change. The `.docx` extension is added automatically if you don't include it.

---


## Creating a New Version

You can create a dated copy of a document from the Client Hub using the **New Version** option. This copies the document with today's date appended to the filename (e.g., `Trust Agreement-20260325.docx`), preserving the original while letting you work on an updated version. If a file with that name already exists, a numeric suffix is added.

---


## The Saved Document

The saved `.docx` is a standard Word document. You can:

- **Open it in Word** (or LibreOffice, Google Docs, etc.) for final review and editing
- **Print it** directly
- **Email it** as an attachment
- **Re-open it in Lily** later to adjust variable values

Re-opening a document in Lily loads the saved variable values from the client's `.lily` file, so you can pick up right where you left off.
