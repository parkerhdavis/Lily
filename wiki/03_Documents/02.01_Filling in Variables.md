# Filling in Variables

The Variable Editor is where you fill in a document's placeholders. It shows a live preview of the document alongside a form with all the variables that need values.

---


## Layout

The Variable Editor has two main areas:

- **Left side — Document Preview** — A rendered preview of the Word document, with variable placeholders highlighted. The preview updates in real time as you fill in values.
- **Right side — Variable Form** — A list of all variables found in the document, each with an input field. Variables appear in the order they first occur in the document.

---


## Variable Types

### Replacement Variables

Most variables are simple text replacements. The template contains a placeholder like `{Client Full Name}`, and you type the value in the form. The preview updates immediately.

**Case handling:** Lily is smart about capitalization. If a template uses the same variable in different cases — say `{Client Full Name}` in one place and `{CLIENT FULL NAME}` in another — Lily applies the right casing automatically:
- ALL CAPS placeholder → your value is uppercased
- all lowercase placeholder → your value is lowercased
- Title Case placeholder → your value is used as-is

### Conditional Variables

Some variables control which text appears in the document based on a yes/no choice. These show up as checkboxes in the form.

For example, a template might contain:
```
{Client Is Single ?? This paragraph is for single clients :: This paragraph is for couples}
```

Toggle the checkbox, and the preview switches between the two text options.

### Date Variables

If a variable has been defined as a date type in the template's [variable schema](wiki/04_Templates/03.02_Variable%20Schemas.md), it shows a date picker instead of a plain text field.

### Currency Variables

Similarly, currency-type variables show a field with a `$` prefix for clearer entry.

---


## Searching Variables

If a document has many variables, use the search field at the top of the variable form to filter (`Ctrl+F` to focus). Only matching variables are shown.

---


## Contact-Bound Variables

Some variables are filled automatically by [contact role assignments](wiki/02_Client_Management/01.02_Contacts.md). These show a link icon and the role name (e.g., "Healthcare POA Agent") so you can see where the value is coming from.

Templates use dot notation to reference contact properties — for example, `{Healthcare POA Agent.full_name}` pulls the full name from the contact assigned to that role.

You can still edit these values manually — the contact binding just provides the initial values. Per-document role overrides let you assign a different contact for a specific document without changing the global assignment.

---


## Interacting with the Preview

Click any highlighted variable in the document preview to scroll to its corresponding field in the sidebar. This is a quick way to navigate when you see a placeholder you want to fill in.

---


## Required Variables

If a variable has been marked as required in the template's [variable schema](wiki/04_Templates/03.02_Variable%20Schemas.md), it shows a "Required" badge. This is a visual indicator — Lily won't prevent you from saving with empty required fields.

---


## Saving

Press `Ctrl+S` (or `Cmd+S` on Mac) to save. Lily writes the variable values into the Word document. An "Unsaved" badge appears in the header whenever you have changes that haven't been saved yet.

If **autosave** is enabled (the default), Lily automatically saves your changes 2 seconds after you stop typing. You can toggle autosave in [App Settings](wiki/06_Settings/05.00_App%20Settings.md).

The saved `.docx` is a standard Word document — you can open it in Word, LibreOffice, or any other word processor for final editing.

---


## Undo and Redo

Made a mistake? Use:
- `Ctrl+Z` — Undo
- `Ctrl+Y` or `Ctrl+Shift+Z` — Redo

Lily groups rapid consecutive changes (like typing a name) into a single undo step, so undoing doesn't go letter by letter.
