# Frequently Asked Questions

Common questions about Lily.

---


## General

### What is Lily?

Lily is a desktop application for legal document drafting and client management. It takes Word document templates with placeholder variables, lets you fill in client-specific information through a form, and produces completed `.docx` files ready for attorney review.

### What platforms does Lily support?

Lily runs on:
- **Windows** (Windows 10 version 1809 and later)
- **Linux** (Ubuntu, Fedora, and most modern distributions)
- **macOS** (10.15 Catalina and later)

### Is Lily free?

Yes. Lily is free and open-source under the GPL license.

### Does Lily require an internet connection?

No. Lily runs entirely offline. Your data stays on your computer.

### Does Lily collect any data?

No. Lily does not phone home, collect telemetry, track usage, or send data anywhere.

## Documents and Templates

### What file formats does Lily work with?

Lily works with **Word documents (`.docx`)**. Templates are `.docx` files with variable placeholders, and the completed documents are standard `.docx` files.

Lily does not work with `.doc` (old Word format), PDF, or other document formats.

### Can I edit the completed document in Word?

Yes. The `.docx` files Lily produces are standard Word documents. Open them in Word, LibreOffice, Google Docs, or any other word processor for final editing, formatting adjustments, or printing.

### Do I need Word installed to use Lily?

No. Lily reads and writes `.docx` files directly — it doesn't rely on Word being installed. However, you'll probably want a word processor available for final review and polishing.

### How do I create a template?

Two options:

1. **In Word:** Create a `.docx` document and type variable placeholders in curly braces: `{Client Full Name}`, `{Date of Birth}`, etc. Save it in your templates folder.
2. **In Lily:** Use the [Template Authoring](wiki/04_Templates/03.01_Template%20Authoring.md) tool. Start with a completed document and visually replace client-specific text with variables.

See [Understanding Templates](wiki/04_Templates/03.00_Understanding%20Templates.md) for the full guide.

### What happens to my original template when I use it for a client?

Nothing — Lily copies the template into the client folder. The original template is never modified.

### Can I use the same variable name across different templates?

Yes, and you should. If multiple templates use `{Client Full Name}`, filling it in for one document automatically fills it in for all of them (within the same client folder). This is one of Lily's key features.

## Variables

### What's the variable syntax?

**Replacement variables:** `{Variable Name}` — replaced with the text you enter.

**Conditional variables:** `{Label ?? text if yes :: text if no}` — shows one of two text options based on a checkbox.

See [Understanding Templates](wiki/04_Templates/03.00_Understanding%20Templates.md) for details and examples.

### Are variables case-sensitive?

Variable *matching* is case-insensitive — `{Client Name}` and `{CLIENT NAME}` are the same variable. But the *casing of each occurrence* controls how the value is displayed: all caps placeholders get uppercased values, and so on.

### A variable in my template isn't showing up. What's wrong?

Word sometimes splits text across internal XML elements, which can prevent Lily from recognizing a variable. Try deleting the placeholder in Word and retyping it in one continuous action (don't pause or click away mid-variable). See [Troubleshooting](wiki/08_FAQ_and_Support/07.01_Troubleshooting.md) for more help.

### What's a conditional variable?

A conditional variable lets the document include different text based on a yes/no choice. For example:

```
{Client Is Married ?? my spouse and I :: I}
```

In the Variable Editor, this appears as a checkbox. Checked → "my spouse and I"; unchecked → "I".

## Client Data

### Where is my client data stored?

Client data is stored in a `.lily` file inside the client folder. This file contains variable values, contacts, and document metadata. It's plain JSON — you can inspect it in a text editor.

### Can I share client data between computers?

Yes. The client folder (including the `.lily` file and documents) can be copied, synced via cloud storage, or shared over a network. You can also use the **Export/Import** feature in the Client Hub to transfer client data as a JSON file.

### Can I back up my data?

Your data lives in regular files and folders — back them up however you normally would. The client folders contain everything: `.lily` files and `.docx` documents.

App settings and questionnaire definitions are stored in your OS config directory (see [App Settings](wiki/06_Settings/05.00_App%20Settings.md) for the exact location).

## The Questionnaire

### What is the questionnaire?

The questionnaire is a structured intake form that walks you through gathering client information section by section. It maps directly to template variables, so filling in the questionnaire is equivalent to filling in variables — just in a more guided format.

### Can I customize the questionnaire?

Yes. Questionnaire definitions are editable from the [Pipeline](wiki/05_Pipeline/04.01_Managing%20Questionnaires.md). You can create new definitions, modify existing ones, or duplicate them as starting points.

## Technical

### What technologies is Lily built with?

- **Tauri 2.0** — Desktop framework (Rust backend)
- **React 19** — UI framework
- **TypeScript** — Type safety
- **Bun** — Runtime and bundler
- **Tailwind CSS + daisyUI** — Styling

### Is Lily open-source?

Yes. Source code is available on [GitHub](https://github.com/parkerhdavis/Lily).

### Why is the installer unsigned?

Code signing certificates are expensive. As a free, open-source project, Lily currently doesn't have code signing. This means you may see security warnings on first install (see [Installation](wiki/01_Introduction/00.02_Installation.md) for how to proceed).
